# CM — Configuration Management

Configuration management is GitOps. Every change to infrastructure or content reaches `main` through a commit that triggers the OPA gate, the Terraform plan, the apply, the canonical inventory build, the Sigstore signing, and the publish. The default workflow for the sole operator is **direct push to `main`**; pull requests are used only when collaboration or staged review is desired (rare). The commit (or PR description, when a PR is used) is the change request, the review when applicable, the approval, and the audit record.

The **Significant Change Notification (SCN) category** for any change is **Routine Recurring by default; no label is required for routine work**. The operator upgrades the categorization explicitly only when the change meets Adaptive or Transformative criteria per `SCN-CSO-EVA`, by including a line like `SCN-Type: adaptive` or `SCN-Type: transformative` in the commit message (or in the PR description, if a PR is used). A fourth value, `SCN-Type: emergency`, is reserved for emergency-change scenarios per `SCN-CSO-EMG` (see [`docs/incident-response.md`](../incident-response.md) for procedures and retroactive-documentation requirements). The CI workflow validates the tag on every push to `main` and every PR — it fails the build only on a malformed value (a recognized `SCN-Type:` line with a value that isn't one of the four recognized values). Missing tags are treated as Routine Recurring per the default. Audit records of these evaluations are the git history (`SCN-CSO-MAR`). Adaptive and Transformative changes — which produce a security impact analysis — additionally get a durable, structured record in [`docs/scn/`](../scn/): a per-change SCN record carrying the `SCN-CSO-INF` fields plus a copy of the SIA, and a row in the machine-and-human-readable register [`docs/scn/scn-register.csv`](../scn/scn-register.csv) (`SCN-CSO-HRM`), retained for at least 12 months (`SCN-CSO-HIS`).

The canonical inventory at `/.well-known/ksi-signal.json` is the authoritative system inventory (CM-8 family). It is generated automatically from Terraform state, Lambda package locks, and content hashes — see [`docs/ksi-signal.md`](../ksi-signal.md). Each component carries a normalized identifier (PURL/ARN/SHA-256/HBOM ref), a security category (per FIPS 199), and an information-flow tag (per `MAS-CSO-FLO`).

**20x rule integration.** Significant Change Notifications (full SCN-* family — `SCN-CSO-MAR`/`INF`/`HIS`/`HRM`/`ARI`/`NOM`/`EMG`/`EVA` plus the Adaptive/Routine-Recurring/Transformative tier rules; transformative timeframes are 30/10/5 business days per `SCN-TRF-NIP`/`NFP`/`NAF`). Minimum Assessment Scope (`MAS-CSO-IIR`/`FLO`/`TPR`/`MDI` — third-party AWS resources documented per `MAS-CSO-TPR`). KSI-CMT (Change Management).

**Review cadence.** Continuous; effectiveness review monthly via the no-out-of-band-commits check on `main`.
